Catalyst Backtesting Engine

01What it does

A strategy graph + a time window in, a full performance report out.

Graph interpreter

Parses Catalyst nodes/edges. Root actions fire at t0; signal → action fires on a rising edge and re-arms; action → action chains run sequentially.

Tick-loop simulator

One candle = one tick. Each tick accrues yield, applies funding, checks liquidations, evaluates signals, executes triggered actions, records equity.

Execution models

Spot swaps (EVM + Hyperliquid), leveraged perps with 8h funding & liquidation, and fixed-APY yield — all with modeled gas/fees/slippage.

Accurate PnL

Spot cost-basis tracking for realized PnL, perp mark-to-market, liquidations recorded as synthetic trades so the log and metrics reconcile.

Metrics

Total return, max drawdown, Sharpe (clamped), win rate, fees — plus an equity curve, trade log, and lifecycle events.

Robustness

Atomic execution (validate before mutating), insufficient-funds → skip + warning, per-action try/except so one bad node never aborts the run.

03Persistence layer (post-MVP data layer)

Opt-in via DATABASE_URL; falls back to a parquet cache so the demo stays zero-ops.

Read-through caching

A backtest serves cached candles/funding from the store and fetches only the missing time gaps from the providers, then persists them. Each candle is fetched once and reused across any overlapping range.

Coverage + staleness

Coverage segments distinguish "fetched but empty" from "never fetched". A freshness horizon (now − interval) always refreshes the still-forming tail. The set arithmetic is pure and unit-tested.

Schema + migrations

Alembic creates candles / funding / coverage; promotes to Timescale hypertables when the extension exists, degrades to plain Postgres (RDS) otherwise.

Backfill CLI

python -m app.data.backfill … pre-warms symbols/intervals through the same gap-fill path. Local dev runs Postgres via docker-compose.

04Engineering challenges solved

The interesting parts — design calls and real bugs hit along the way.

Repeating strategies

Signals fire on a rising edge and re-arm on reset, so a rung fires once per threshold crossing — not every tick. → correct round-trip behavior.

Geo-blocked data

Binance's main API returns HTTP 451 in some regions. → provider probes & falls back across data-api.binance.vision → binance.us.

Yield-only & Sharpe blow-ups

No-price strategies had no timeline; near-zero volatility gave absurd Sharpe. → synthesize a timeline; clamp Sharpe to ±100.

New AWS account blocked

Free Plan blocks App Runner & restricts RDS. → upgrade to paid plan (surfaced clearly, not silently worked around).

App Runner egress trap

A VPC connector routes all egress through the VPC → no internet for data fetches. → private subnets through a cheap NAT instance.

Terraform data-source loop

aws_subnets began including the subnets it was creating, placing the NAT in a private subnet. → restrict with the default-for-az filter.
