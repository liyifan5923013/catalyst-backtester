Getting started

Catalyst Backtester answers one question: if this strategy had run in the past, what would have happened?

  1. Load an example from the dropdown (easiest), or paste your own Catalyst graph JSON.
  2. Pick a date range and granularity (15m, 1h, 4h, 1d). All dates are UTC.
  3. Click Run backtest. The engine fetches historical prices, simulates each candle as one tick, and returns metrics + charts.
  4. Optionally enable comparison to run a second window (previous period or year-over-year) and see deltas.

Strategy graphs

A Catalyst graph is JSON with nodes (actions and signals) and edges (control flow).

How the engine executes a graph

  • Root actions (no incoming edge) run once at the start of the backtest.
  • Signal → action edges fire when a condition turns true (rising edge), then re-arm when it goes false again — so ladders repeat correctly.
  • Action → action edges run the next action immediately after the previous one finishes.
  • Nodes with "enabled": false are skipped.
{
  "nodes": [
    { "id": "buy", "kind": "action", "subtype": "swap",
      "config": { "from_asset": "USDC", "to_asset": "ETH", "amount": "100", "chain": "base" },
      "enabled": true }
  ],
  "edges": []
}

Supported actions

Swap (spot)

chain: base / EVM — priced from Binance. chain: hyperliquid — Hyperliquid spot. chain: equity — US stocks (e.g. AAPL) from Yahoo Finance. Buys use USD notional; sells use token quantity or "all".

Perp order

Hyperliquid perpetuals only. Supports open/add/close with leverage, funding every 8h, and liquidation when margin is exhausted.

Yield deposit / withdraw

Aave-style USDC yield with a flat configurable APY (default 5%). No live protocol rates in MVP.

Price threshold (signal)

Boolean condition on price each tick: operator (< or >) and threshold. For stocks, add "market": "equity" to the signal config.

Running a backtest

Sidebar / Setup tab

  • Example strategy — loads a pre-built graph into the JSON editor.
  • Analysis period — start/end dates (UTC badge shown).
  • Granularity — one candle = one simulation tick.
  • Initial capital — starting USDC balance (default $10,000).
  • Strategy graph (JSON) — must be valid JSON; invalid graphs block the run button.

On desktop, use ‹ Hide to collapse the inputs panel and focus on results. On mobile, use the bottom Setup / Results tabs.

First run may take a few seconds while market data is fetched. Subsequent runs over the same range are faster when data is cached.

Period comparison

Enable Compare to another period to run the same strategy twice and see how the current window stacks up.

  • Previous period — immediately preceding window of equal length (recommended for Hyperliquid strategies).
  • Year-over-year — same calendar dates one year earlier (works best when enough history exists).
  • Custom — pick any comparison start/end.

Metric cards show ▲/▼ deltas vs the comparison (green/red depends on whether higher is better). The equity chart overlays both periods aligned by elapsed time. If the comparison period has no data, you still get primary results plus a yellow warning.

Reading results

  • Summary & recommendations — AI narrative when configured; otherwise a rule-based summary. Both reflect comparison trends when enabled.
  • Metric cards — hover the i dot on each label for a plain-English definition.
  • Portfolio equity chart — green = current period; dashed amber = comparison. Blue dashed line = ETH reference price. Tap Fullscreen for detail.
  • Events — warnings (skipped actions, liquidations) and errors; timestamps in UTC.
  • Trades — every executed swap, perp, or yield action with fees and realized PnL.

Metrics glossary

MetricMeaning
Final equityMark-to-market account value at the end (cash + positions).
Total return(Final equity − initial capital) ÷ initial capital, as a percentage.
Max drawdownLargest peak-to-trough drop in equity during the period. Lower is better.
Sharpe ratioRisk-adjusted return (annualized mean ÷ volatility of per-candle returns). Higher is better.
TradesCount of executed trades, including forced liquidations.
Total feesSum of modeled trading fees, slippage, and gas.
Win rateShare of closing trades with positive realized PnL.

AI summary

After a run, the app requests a short narrative plus 2–4 recommendations from an LLM (when an API key is configured on the server). The summary uses your metrics and, if comparison is on, period-over-period trends.

If the LLM is unavailable, a deterministic fallback still produces a summary — the badge shows AI generated vs Auto (heuristic).

Data sources

  • Binance — crypto spot prices for EVM chains (ETH, etc.).
  • Hyperliquid — spot/perp candles and 8h funding rates. History limited to ~5000 recent candles.
  • Yahoo Finance — US equity OHLCV (no API key). Alpha Vantage is used as fallback when configured.
  • Stablecoins — USDC/USDT/DAI pegged at $1.
Hyperliquid spot/perp history is relatively short. For comparisons on HL strategies, prefer Previous period over year-over-year.

Mobile

On phones, the app uses a dedicated mobile layout with Setup and Results tabs, a sticky run button, and touch-friendly controls. Add to your home screen (Safari → Share → Add to Home Screen) for an app-like experience.

FAQ

“No market data returned for the requested range”

Check that your date range has data for the symbols and chain in the graph. Hyperliquid history is limited; try a recent range or use Binance/EVM or equity examples. For comparison failures, the primary run may still succeed — look for the yellow warning.

Why are there warning events but the backtest still finished?

Insufficient balance or margin skips an action and logs a warning instead of crashing the whole run. Review Events for details (e.g. “Insufficient USDC”).

What timezone are dates in?

All dates and chart times are UTC. The UTC badge on the date picker is your reminder.

Can I backtest stocks?

Yes — use "chain": "equity" on swap actions (e.g. buy AAPL with USDC). Load the “US Equity - Buy $1000 of AAPL” example.

Is this live trading?

No. This is simulation only — no wallets, no real orders. Results include modeled fees and slippage, not exact exchange execution.
